# 全新 Windows 电脑一键部署

## GitHub 下载

打开 https://github.com/yrupeechalco-cell/medical-literature-tracker-hermes/releases/latest，
在 **Assets** 中下载 `medical-literature-tracker-full-windows-<版本号>.zip`。
不要使用轻量源码包，也不要使用 GitHub 自动生成的 `Source code` 压缩包。

面向普通用户的逐步下载说明见 [`../新电脑下载与安装.md`](../新电脑下载与安装.md)。
预装软件、账号权限和网络条件见 [`../预装软件与环境清单.md`](../预装软件与环境清单.md)。

适用系统：Windows 10 或 Windows 11。电脑不需要预装 Python、Git、Node.js、
Hermes、Codex 或 WSL2。完整包内含固定版本的 Hermes、CC Switch 和 uv；仍需要
可访问 DeepSeek、PubMed 和飞书的网络。

## 部署前准备

1. 一个可用的 DeepSeek API Key。
2. 飞书账号，以及创建或安装企业自建机器人的权限。
3. 至少 5 GB 可用磁盘空间。
4. 解压完整安装包，不能只把 `INSTALL_WINDOWS.cmd` 单独复制出来。

飞书桌面端和 Obsidian 不在安装包内。它们是成熟的独立软件，可以提前从官方网站
安装；部署程序只检测并提示，不复制或修改它们。

## 一键安装

双击根目录的 `INSTALL_WINDOWS.cmd`。

安装器会自动完成：

1. 将系统复制到 `%LOCALAPPDATA%\MedicalLiteratureTracker`。
2. 完整包直接使用内置的 Hermes 官方 `v2026.6.5` release 源码；轻量包缺少该载荷时才联网下载。
3. 使用固定版本 uv 创建隔离 Python 3.11，只安装 Hermes core、cron 和飞书组件；
   严格按 Hermes 自带 `uv.lock` 安装，不安装本系统不需要的浏览器、语音或 Node.js 工具。
4. 隐藏输入并保存 DeepSeek API Key。
5. 将运行模型固定为 `deepseek-v4-pro`，Provider 固定为 `deepseek`。
6. 打开 Hermes 消息平台向导。选择飞书，推荐使用二维码自动创建应用。
7. 安装并启动 Hermes gateway，同时创建每 5 分钟一次的恢复检查。
8. 等待用户向飞书机器人发送第一条私聊消息。
9. 创建每天 `07:30` 的医学文献追踪任务。
10. 执行首次真实采集、Hermes 摘要和飞书投递测试。

CC Switch 会以便携版安装到 `%LOCALAPPDATA%\CCSwitchPortable`，用于管理配置，
但不参与定时任务执行。医学文献追踪运行时仍只依赖 Hermes Agent。

完整包约 114 MB。首次安装仍需联网下载隔离 Python 3.11 和锁定的 Python wheel；
飞书、Obsidian、DeepSeek 服务和文献 API 也需要网络。本包不是完全离线镜像。

DeepSeek Key、飞书 Secret、聊天 ID 不会写入项目目录、安装包或 GitHub。

## 飞书步骤

当 Hermes 向导打开时，选择 `Feishu/Lark`。二维码模式会在浏览器打开飞书开放平台，
由 Hermes 自动创建并配置应用。扫码和授权必须由飞书账号本人完成。

gateway 启动后，安装器会提示向新机器人发送任意一条私聊消息。Hermes 记录该会话
后，定时任务才会绑定投递目标。这样安装包内不需要保存任何人的聊天 ID。

## 自定义安装盘

默认项目和文献数据位于系统盘。若希望放到其他磁盘，在 PowerShell 中执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\deploy\windows\install.ps1 `
  -InstallDir "F:\MedicalLiteratureTracker" `
  -HermesHome "F:\Hermes"
```

数据库、原始 API 响应、报告和日志都会跟随 `InstallDir`，Hermes 配置与密钥跟随
`HermesHome`。

## 检查与恢复

- 双击 `CHECK_WINDOWS.cmd`：检查 Hermes、模型锁、工具隔离、任务、skill 和数据库。
- 重复双击 `INSTALL_WINDOWS.cmd`：安全修复缺失文件或未完成配置，不删除历史数据。
- gateway 每 5 分钟自动执行一次幂等启动检查，异常退出后可自动恢复。
- 双击 `UNINSTALL_WINDOWS.cmd`：移除追踪任务和 skill，默认保留文献数据。

## 安全边界

- 运行时仅使用 Hermes Agent 的 `terminal` 和 `file` 工具集。
- 定时任务不调用 Codex、OpenClaw、其他 agent、subagent 或 delegate。
- 完整包内置固定 release tag 的 Hermes 官方源码 ZIP，不依赖系统 Git，也避免
  上游 `main` 变化和 Windows 换行转换造成的 checkout 冲突。
- 本系统用于科研筛选，不提供临床诊断或治疗建议。
