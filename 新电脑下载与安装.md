# 新电脑下载与安装

适用于 Windows 10 或 Windows 11。新电脑不需要预装 Python、Git、Node.js、
Hermes、Codex 或 WSL2。

## 1. 下载正确的文件

打开最新版本页面：

https://github.com/yrupeechalco-cell/medical-literature-tracker-hermes/releases/latest

在页面底部展开 **Assets**，下载文件名包含 `full-windows` 的完整包：

`medical-literature-tracker-full-windows-0.3.1.zip`

也可以直接下载当前稳定版：

https://github.com/yrupeechalco-cell/medical-literature-tracker-hermes/releases/download/v0.3.1/medical-literature-tracker-full-windows-0.3.1.zip

不要下载以下文件作为新电脑安装包：

- `medical-literature-tracker-hermes-0.3.1.zip`：轻量源码包，需要自行准备更多依赖。
- GitHub 自动生成的 `Source code (zip)` 或 `Source code (tar.gz)`：不含完整安装载荷。

仓库当前为私有仓库时，下载者必须先登录已获授权的 GitHub 账号。

## 2. 安装前准备

完整准备清单见 [`预装软件与环境清单.md`](预装软件与环境清单.md)。

准备以下内容：

1. 可用的 DeepSeek API Key。
2. 飞书账号，以及创建或安装企业自建机器人的权限。
3. 稳定网络连接。
4. 至少 5 GB 可用磁盘空间。

飞书桌面端可以提前从 https://www.feishu.cn/download 安装。Obsidian 是可选软件，
不影响文献追踪核心功能。

## 3. 解压并安装

1. 下载完成后，右键 ZIP 文件并选择“全部解压”。
2. 打开解压后的完整文件夹。
3. 双击 `INSTALL_WINDOWS.cmd`，不要只把该文件单独复制到其他位置。
4. 按提示输入 DeepSeek API Key。
5. Hermes 向导出现后选择 `Feishu/Lark`，推荐二维码方式。
6. 在飞书中完成扫码和应用授权。
7. gateway 启动后，向新机器人发送任意一条私聊消息。
8. 等待首次采集、摘要和投递测试完成。

如果 Windows 显示来源提示，请先确认文件确实来自上述 GitHub Release，再选择运行。
不要运行来自网盘、聊天附件或未知仓库的同名安装包。

## 4. 安装后检查

双击 `CHECK_WINDOWS.cmd`。以下项目均显示 `[OK]` 才表示安装完成：

- Hermes 路径
- DeepSeek Provider 与 `deepseek-v4-pro` 模型锁
- `terminal`、`file` 工具隔离
- 每日任务
- tracker skill
- SQLite 历史数据库

默认每天 `07:30` 执行。重复运行 `INSTALL_WINDOWS.cmd` 可以修复安装，不会删除
已有文献历史。

## 5. 升级旧版本

下载新的 `full-windows` ZIP，解压到一个新文件夹，再运行其中的
`INSTALL_WINDOWS.cmd`。安装器会更新程序并保留原 SQLite 文献历史、去重账本和配置。

如果旧版本使用了自定义安装盘，升级时继续使用原来的 `InstallDir` 和 `HermesHome`，
具体命令见 [`docs/全新电脑一键部署.md`](docs/全新电脑一键部署.md)。

## 6. 当前安装包校验值

`medical-literature-tracker-full-windows-0.3.1.zip`

SHA-256：

`192dbbb8a668f2c27a4f702ef29c6539434e6e5b8a4ef2b7dcbd5813b9304c70`

PowerShell 校验命令：

```powershell
Get-FileHash .\medical-literature-tracker-full-windows-0.3.1.zip -Algorithm SHA256
```

输出应与上面的 SHA-256 完全一致。
